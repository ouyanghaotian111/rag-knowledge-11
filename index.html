三步合一，让 LLM 拥有"开卷考试"的能力

一句话定义

RAG 是一种在 LLM 生成回答之前，先从外部知识库中检索相关信息，将检索到的内容作为上下文注入 Prompt，从而增强生成质量的技术架构。

它解决了大模型"知识截止"和"幻觉"两大核心痛点，让 AI 能够基于最新的、专属的、可信的知识来回答问题。

💡 产品经理的类比

纯 LLM = 闭卷考试（凭记忆答题）
RAG = 开卷考试（可以翻书找答案再作答）

MODULE 02

为什么需要 RAG？

纯大模型面临的四大痛点

🧊

知识截止 (Knowledge Cutoff)

模型训练数据有截止日期，无法获知最新信息。你的企业内部数据它更无从得知。

👻

幻觉问题 (Hallucination)

模型会"一本正经地胡说八道"，生成看似合理但实际错误的内容，在企业场景可能导致严重后果。

🔒

私域知识缺失

企业内部文档、流程、产品手册、FAQ 等私域知识，大模型在训练时从未见过。

💰

微调成本高昂

Fine-tuning 需要大量标注数据和 GPU 算力，且每次知识更新都需重训。RAG 只需更新知识库。

MODULE 04

核心技术详解

产品经理需要理解的关键技术概念

🔢

Embedding（向量嵌入）

将文本转换为高维数值向量。语义相近的文本，其向量在空间中距离更近。这是 RAG 实现语义搜索的基石。

  • 常用模型：OpenAI text-embedding-3, BGE, M3E, Jina
  • 向量维度通常为 768 / 1024 / 1536 维
  • 中文场景推荐 BGE / M3E 等本土化模型
  • 产品关注：模型选型直接影响检索准确率
✂️

Chunking（文档切分）

将长文档切成适合检索和 LLM 上下文窗口的小段。切分策略直接影响检索质量。

  • 固定长度切分：简单但可能割裂语义
  • 语义切分：按段落/标题/句子自然断开
  • 递归切分：先大后小，逐层细化
  • Overlap 重叠：相邻块重叠 10-20% 防信息丢失
  • 典型 Chunk 大小：200-1000 tokens
🔍

检索策略

如何从海量知识块中精准找到与问题最相关的内容，是 RAG 系统的核心挑战。

  • 语义检索：基于向量相似度，理解语义
  • 关键词检索：BM25 等传统方式，精确匹配
  • 混合检索：语义+关键词，取长补短 ⭐推荐
  • 重排序 Rerank：用模型对初筛结果精排
  • Top-K 通常取 3-10 条，需权衡准确率和上下文长度
📝

Prompt Engineering

将检索结果和用户问题组装成 LLM 能高效利用的 Prompt，直接决定回答质量。

  • 角色设定：明确 AI 的身份和回答风格
  • 上下文注入：将检索内容标记为参考资料
  • 约束指令：要求"仅基于提供的资料回答"
  • 格式控制：指定输出格式（列表/表格/步骤）
  • 兜底策略：找不到答案时如何回复

MODULE 06

产品经理的 RAG 关注点

从产品视角看 RAG 系统建设的核心决策

1

📊 需求分析框架

明确三个核心问题：知识源是什么？用户是谁？质量要求如何（容错率/响应时间）？

2

🏗️ 技术方案评审

能与工程师讨论：Embedding 选型、Chunk 策略、检索方式、向量库选择、Prompt 模板。

3

🎯 效果评估体系

建立量化标准：准确率、召回率、忠实度、用户满意度，以及 Bad Case 归因分析流程。

4

💰 成本控制

关注 API 成本（Embedding + LLM）、向量库存储成本、知识库维护人力成本。做好 ROI。

5

🔄 迭代优化策略

建立"失败→归因→优化"飞轮：检索没召回？召回不相关？LLM 没用好上下文？分层定位。

6

🛡️ 安全与合规

数据隐私保护、输出审核机制、权限管控（谁能访问哪些知识）、审计日志。

MODULE 09

进阶优化策略

从基础 RAG 到生产级 RAG 的进化路径

🔀 Query Rewriting

对用户原始问题进行改写/扩展/分解，提升检索命中率。例如将口语化问题转为精确查询，或将复杂问题拆解为多个子问题。

🏷️ Metadata Filtering

为知识块附加元数据（来源、时间、类别等），检索时结合元数据过滤，缩小搜索范围，提升相关性。

🔄 Reranking 重排序

初筛后用 Cross-Encoder 等重排序模型对结果精排，显著提升 Top-K 结果的质量。常用模型：Cohere Rerank、BGE-Reranker。

🧠 HyDE 假设文档

先让 LLM 根据问题生成一个"假设回答"，用这个回答去做向量检索，往往比直接用问题检索效果更好。

📊 Parent-Child Chunk

小 Chunk 用于精确检索，命中后返回其父级大 Chunk 作为上下文，兼顾检索精度和上下文完整性。

🔗 Graph RAG

结合知识图谱进行检索增强，利用实体关系进行多跳推理，适合需要关联分析的复杂问答场景。

💾 多级缓存

对高频相似问题缓存检索结果和回答，减少重复计算，降低成本并提升响应速度。

🔁 Self-RAG 自省

让 LLM 自我判断是否需要检索、检索结果是否有用、回答是否忠实，形成自适应的检索-生成循环。

📐 Agentic RAG

将 RAG 与 Agent 结合，AI 自主决定检索策略、选择工具、多步推理，处理更复杂的任务链路。

知识测验 🧪

检验你对 RAG 的理解程度

Q1RAG 中的 "R" 代表什么？
A. Reinforcement（强化）
B. Recursive（递归）
C. Retrieval（检索）
D. Reasoning（推理）
✅ 正确答案：C. Retrieval（检索）。RAG = Retrieval-Augmented Generation，即检索增强生成。核心思想是在生成回答前，先从外部知识库检索相关信息。
Q2以下哪项 NOT 是 RAG 相比纯 LLM 的优势？
A. 减少幻觉，提供可溯源的回答
B. 提升模型本身的推理能力
C. 可以接入最新的私域知识
D. 无需重新训练即可更新知识
✅ 正确答案：B。RAG 增强的是模型能获取的知识范围，而非模型本身的推理能力。推理能力是模型固有能力，需要通过训练/微调来提升。
Q3在 RAG 系统中，Embedding 的主要作用是？
A. 生成最终回答
B. 切分文档
C. 将文本转为向量以实现语义搜索
D. 存储文档元数据
✅ 正确答案：C。Embedding 模型将文本转换为高维数值向量，使得语义相近的内容在向量空间中距离更近，从而实现基于语义的相似性搜索。
Q4作为 PM，以下哪个 RAG 评估指标最能反映"AI 是否在编造内容"？
A. 检索召回率 (Recall)
B. 响应延迟 (Latency)
C. 回答相关性 (Relevance)
D. 回答忠实度 (Faithfulness)
✅ 正确答案：D. 回答忠实度 (Faithfulness)。这个指标衡量的是 LLM 的回答是否忠于检索到的参考资料，如果模型"脱离上下文自由发挥"，忠实度就会很低。
Q5混合检索（Hybrid Search）结合了哪两种检索方式？
A. 图检索 + 向量检索
B. 关键词检索（BM25）+ 向量语义检索
C. SQL 查询 + 全文搜索
D. 图片检索 + 文本检索
✅ 正确答案：B。混合检索结合了传统关键词检索（如 BM25，擅长精确匹配）和向量语义检索（擅长理解语义），两者取长补短，是目前最推荐的检索策略。
